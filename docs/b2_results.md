# Phase B2 결과: 엔진 컴포넌트 ablation (leave-one-out, 2026-07-21)

**질문:** B1은 유도 연산자의 기여를 보였다. 그렇다면 나머지 성능은 엔진 골격의
어느 컴포넌트에서 오는가? selector(uniform)·연산자 풀(full)을 고정하고 골격
컴포넌트를 하나씩 제거해 한계 기여를 측정.

- 실행: `experiments/b2_run.py` → `outputs/b2_component_ablation.jsonl`
  (test pool, 5 arm × 3 규모 × 3 TW × 5 인스턴스 × 5 반복 = 1125 run)
- 요약: `experiments/b2_summary.py` → `outputs/b2_summary.txt`
- 재현성: `none` arm(전부 유지)은 기존 `GILS-uniform-1000`과 소수점까지 동일.
- 엔진 토글: `VaaQRLConfig.use_descent/use_sa_acceptance/use_restart`
  (전부 True면 표준 엔진과 바이트 동일), init 제거는 random 초기해 주입.

## 해석 — 성능의 출처 분해

컴포넌트 중요도 순위(제거 시 악화 크기):

1. **descent (best-improvement 지역탐색): +0.156%.** 단일 최대 기여, 모든
   범위에서 유의. 엔진 품질의 핵심 동력.
2. **kick-restart: +0.069%.** 두 번째로 중요, 유의. 정체 탈출에 기여.
3. **VAA 초기해: +0.022%, 분할 시 무의미.** random 초기해로 시작해도 거의
   동일 수렴 → **초기해는 사실상 무관**(init_sensitivity의 ±0.33% 결론 재확인,
   학습 기반 초기해(direction B) 무용성과 정합).
4. **SA 수용: −0.026%(제거가 오히려 미세 개선).** M/L에서 greedy가 SA보다
   근소하게 우수. **강한 descent + 유도 연산자 + restart가 이미 작동하므로,
   확률적 수용은 품질 동력이 아니다.** (단일 인스턴스에선 greedy가 크게 막히는
   경우도 있으나 test pool 평균에서는 상쇄.) 직접 재계산으로 부호 확인.

## B1 + B2 통합 그림 (논문 핵심 분해)

성능은 **결정론적 구조**에서 나온다:

```
VAA 초기해(6.49% gap, S-none)
  └─ descent + restart (엔진 골격)  ← 큰 덩어리 (6.49% → ~0.45%)
       └─ 유도 연산자 g1~g4 (B1)     ← 유의 정련 (0.45% → 0.10%)
```

반면 다음은 **기여하지 않는다**:
- 학습된 연산자 선택 (K1: ≤0.17%p, 방향 전환, 학습 무승),
- 학습/좋은 초기해 (B2 init: 무의미, direction B 사망),
- 확률적 SA 수용 (B2 sa: 동력 아님).

→ 논문 주장: **"품질은 문제 구조를 이용한 결정론적 지역탐색(descent + 유도
연산자 + restart)에서 나오며, 학습(선택·구성)과 확률적 수용은 이 환경에서
기여하지 않는다."** B1·B2가 이를 정량적으로 뒷받침.

## 논문 반영

- 표: "Engine-component ablation" (위 Wilcoxon 표). B1 표와 나란히 배치.
- 그림(선택): 위 분해 트리 또는 컴포넌트별 악화율 막대그래프.
- SA 결과는 정직하게 서술: 제거해도 악화되지 않음 → 방법을 단순화할 여지가
  있으나, 관례상·강건성 위해 유지한다고 각주 처리 가능.
