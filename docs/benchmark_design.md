# 벤치마크 설계: 논문 실험 세팅 대조와 반영

베이스 논문(Shahmardan & Sajadieh 2020) Section 4~5 및 Table 2~7의 실험 세팅을
정리하고, 우리 벤치마크와의 차이·반영 사항을 기록한다.

## 1. 논문의 인스턴스 생성 (Section 4)

- 데이터 기반: Van Belle et al. (2013), I-형(I-shape) 크로스도킹 센터.
- 인접 도어 간 이송 시간 = 1초 (I-형 선형 배치이므로 도어 m,n 거리 = |m − n|).
- DE_i, DL_i ~ U(0, 20)   (진입/출차 시간)
- t_k ~ U(3, 10)          (상품 단위 하역/적재 시간)
- f_idk ~ U(0, 20)        (초기 적재량)
- 크기 표기 = (I, D, M) = (컴파운드, 목적지, 도어).
- 소규모: 컴파운드/목적지/도어 각각 7/10/7 미만 (Table 2: (2,3,2)~(6,9,6)).
- 대규모: (8,10,8)부터 (20,30,20)까지 (Table 4).
- 인스턴스마다 반복 20회. 정확해는 GAMS/CPLEX, 시간제한 10,800초(3시간).

## 2. 시간 예산 공식 (검증됨)

메타휴리스틱 종료 기준:
```
T_budget(s) = ((|I| + |D|) / 2) × |M| × 0.7
```
Table 5 대조 검증: (8,10,8)→50.4s, (20,30,20)→350s, (2,3,2)→3.5s 모두 일치.
E6(동일 시간 예산 비교)에서 이 공식을 채택해 논문 프로토콜과 직접 비교 가능.

## 3. 우리 생성기와의 차이 (중요)

| 항목 | 논문 | 현재 우리 생성기 |
|---|---|---|
| 컴파운드 비율 | **67~80%** (컴파운드 다수, 아웃바운드가 나머지 D−I 충당) | **25~30%** (아웃바운드 다수) |
| |I| 대 |M| | |I| ≈ |M| (종종 동일) | |I| < |M| 여유 큼 |
| |D| 대 |M| | 1.25~1.5배 | 2~3.3배 (목적지 과다) |
| 이송 시간 | I-형, 인접 = 1 (|m−n|) | 100×100 평면 유클리드/10 |
| DE/DL | U(0, 20) | U(1, 5) |
| t_k | U(3, 10) | U(1, 5) |
| f_idk | U(0, 20) | U(0, 20) (동일) |
| 반복 | 20 | 30 (계획) |

가장 중대한 차이는 **컴파운드:아웃바운드 비율의 역전**이다. 부분하역(partial
unloading)은 컴파운드 트럭에서만 일어나므로, 컴파운드가 다수인 논문 체제가
이 메커니즘을 훨씬 강하게 자극한다. 현재 우리 아웃바운드 과다 인스턴스는
논문이 다루는 영역을 대표하지 못한다.

## 4. DBPR (목적지 화물 집중도, Eq. 32)

```
DBPR_d = max_{i∈I} ( Σ_k f_idk t_k / Σ_{d'} Σ_k f_id'k t_k )
```
한 목적지 수요가 특정 컴파운드에 얼마나 몰려 있는지의 척도. 논문 민감도분석
(Table 6,7): DBPR이 높을수록 부분하역의 makespan 개선효과가 커짐(최대 55.9%).
우리 `clustered` 흐름 패턴이 DBPR을 높이는 방향과 대응. 벤치마크가 저→고 DBPR을
포괄하도록 하고 인스턴스별 DBPR을 리포트하면 E7(민감도) 근거가 강해짐.

## 5. 반영 결정

### 확정 (구현)
1. 시간 예산 공식 `paper_time_budget()` 채택 (E6).
2. DBPR 특성값 `dbpr()` 계산 함수 + 인스턴스 리포트.
3. 논문 충실(paper-faithful) 파라미터/레이아웃 모드 추가:
   DE/DL~U(0,20), t_k~U(3,10), I-형 이송(|m−n|). 목적: 우리 VAA/SA-RL 재현이
   논문 Table 2/4 수치를 재현하는지 검증(코드 신뢰성 확보) + 논문 체제 대표.
4. CP-SAT 시간제한을 논문과 정합되게 설정(대규모 3시간 상한 참고).

### 확정: 규모 클래스 논문 정합 재정의 (2026-07-05)
SIZE_CLASSES를 컴파운드 다수(|I|=|M|, |D|=1.5|I|, 컴파운드 비율 2/3)로 교체:

| 클래스 | (I, D, M) | 컴파운드 | 아웃바운드 | 도어 | 근거 |
|---|---|---:|---:|---:|---|
| S | (6, 9, 6) | 6 | 3 | 6 | 논문 Table 2 정확해 가능 최대 |
| M | (12, 18, 12) | 12 | 6 | 12 | 중간 |
| L | (20, 30, 20) | 20 | 10 | 20 | 논문 Table 4 대규모 최대 |
| XL | (30, 45, 30) | 30 | 15 | 30 | 논문 확장 |

전부 |I| = |M| (도어당 컴파운드 1대, 논문 대표 구성), |D| = 1.5|I|.
XXL(200대급, Li et al. 2025 대응)은 후속 확장 옵션으로 보류.

## 6. 재현 검증 계획 (신규)

논문 충실 모드로 소규모 (2,3,2)~(6,9,6) 및 대규모 일부를 생성해, 우리 VAA와
Paper-SA-RL5의 makespan이 논문 Table 2/4 경향과 일치하는지 확인. 일치하면
"우리 재구현이 논문을 충실히 재현" 문장을 논문에 넣을 수 있어 신뢰성 크게 향상.
(주의: 논문 인스턴스 원본은 비공개라 값 자체가 아니라 경향·상대순위 비교.)
